Project Overview

This project is an internal HR portal designed to support two main roles: Admin and Intern. The goal was to centralize key HR workflows such as document management, leave requests, payslip handling, and personal profile settings within a clean, role-based dashboard.

Goals

  • Create a clear dashboard overview for daily HR actions
  • Make tables and statuses readable and easy to scan
  • Support file upload and validation workflows
  • Keep the interface simple for interns (self-service)
  • Ensure consistency across admin and intern experiences

My Role

  • End-to-end UX/UI design in Figma
  • Designed role-based navigation (Admin vs Intern)
  • UI system consistency (spacing, components, tables)
  • Iterated designs with the developer/PM feedback

Key Screens

Selected screens showing the main HR workflows for both roles.

What I Focused On

  • Clarity and scanability (cards, tables, status tags)
  • Role separation without duplicating the entire UI
  • Action-first dashboard layout (quick actions)
  • Consistency across screens (reusable components)
